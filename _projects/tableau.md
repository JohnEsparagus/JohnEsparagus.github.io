## Overview

This parser takes in logical formulas and determines whether they are satisfiable. It handles both propositional logic (e.g. `p`, `q`, `=>`, `/`) and first-order logic (e.g. `∀`, `∃`).

The program's purpose is simple: check if a formula can be true under some interpretation.

## Technical details

The parser analyzes formulas and determines whether there exists an assignment of values that makes them true.

### Propositional logic

The parser looks for connectives like `/` (OR), `/\` (AND), `=>` (implies), and `~` (negation). It evaluates whether a formula can be true with some combination of truth values.

### First-order logic

It identifies quantifiers (`∀`, `∃`) and checks if the formula holds for all or some values of variables. Once the formula is parsed, it tells you if it is satisfiable or not satisfiable.

There are still a few bugs with the first-order logic implementation, but it is not difficult to extend further.

## Example inputs and outputs

**Formula:** `~(p => (q => p))`
**Output:** Not satisfiable. The negation of this implication can't be true.

**Formula:** `(~(p => q) /\ q)`
**Output:** Not satisfiable. This combination can't hold for any truth assignments.

**Formula:** `(~~~p / (q /\ ~q))`
**Output:** Satisfiable. This works with certain truth assignments.

**Formula:** `(p => p)`
**Output:** Satisfiable. This is always true (a tautology).

**Formula:** `(q /\ ~(p / ~p))`
**Output:** Not satisfiable. The negation of a tautology is always false.

## How to use

Clone the repository and run the parser with your own formulas:

```bash
git clone https://github.com/JohnEsparagus/tableau-solver.git
cd tableau-solver
python3 parser.py

cat input.txt
python3 parser.py > output.txt
```
